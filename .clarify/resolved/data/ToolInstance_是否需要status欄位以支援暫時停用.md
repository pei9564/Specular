# 釐清問題

目前的 `ToolInstance` 僅有配置資訊，是否應新增 `status` 欄位以支援「暫時停用」特定工具，而不必將其刪除？

# 定位

ERM：`ToolInstance` Table。
Feature：`Tool設置.feature`。

# 多選題

| 選項 | 描述 |
|--------|-------------|
| A | **不需要**：若不使用直接刪除即可，系統結構越簡單越好。 |
| B | **需要 (Active/Inactive)**：允許停用工具，停用後的工具在 Context Assembly 時不會被注入。 |
| Short | 維持簡單結構，暫不需要 |

# 影響範圍

影響管理員對工具實例的可控性，以及資料庫欄位的擴展。

# 優先級

Low

- 管理功能完備性

---

# 解決記錄

- **回答**：A - 不需要：若不使用直接刪除即可，系統結構越簡單越好。
- **更新的規格檔**：無
- **變更內容**：維持現狀，不新增 status 欄位。
